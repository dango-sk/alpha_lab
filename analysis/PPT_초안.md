## 슬라이드 1: 모델의 목적

### 왜 만들었나
- 이현의 밸류 투자 프로세스를 **정량적 모델로 체계화**
- 종목 선정 시 **주관적 판단의 편향을 줄이고**, 일관된 기준 적용
- 백테스트를 통해 **전략의 유효성을 사전 검증**한 후 실전 운용에 참고

### 무엇을 하는 모델인가
- 매월 1회, KOSPI 유니버스에서 **상위 30개 종목을 자동 선정**
- 선정 기준: 밸류(싼가), 회귀(적정가 대비 괴리), 성장(실적 성장), 차별화(시장 기대 변화)

## 슬라이드 3: 모델 구조 — 4가지 팩터

```
총점 = 밸류(35%) + 회귀(30%) + 성장(20%) + 차별화(15%)
```

| 팩터 | 비중 | 보는 것 | 예시 지표 |
|------|------|--------|----------|
| **밸류** | 35% | 현재 주가가 싼가 | PER, PBR, EV/EBITDA, PCF (현재+Forward) |
| **회귀** | 30% | 적정가 대비 괴리가 있나 | PBR-ROE 회귀, EV/IC-ROIC 회귀, F.PER-이익성장 회귀, F.EV/EBIT-EBIT성장 회귀 |
| **성장** | 20% | 실적이 성장하고 있나 | 매출 성장률 (현재+Forward) |
| **차별화** | 15% | 시장 기대가 올라가고 있나 | EPS 컨센서스 변화율 (모멘텀) |

### 회귀 팩터란?
- 단순히 "싸다"가 아니라 **"같은 수준의 수익성(ROE) 대비 싸다"**를 측정
- 예: ROE 15%인 A종목과 B종목이 있을 때, A의 PBR이 1.0이고 B가 0.7이면 → B가 회귀선 아래 → **"적정가보다 저평가"**

## 슬라이드 5: 회귀 팩터의 역할 — 있을 때 vs 없을 때

### 3가지 전략 비교

| 지표 | 기존전략 (회귀 30%) | 회귀 제외 | 회귀 Only |
|-----|-------------------|----------|----------|
| **누적수익률** | **167.1%** | 93.5% | 140.1% |

| **Sharpe** | **0.90** | 0.69 | 0.90 |

- 회귀 팩터가 모델에서 **가장 중요한 알파 소스**
- 회귀 Only도 Sharpe 0.90으로 단독으로도 강력

## 슬라이드 7: 회귀 팩터의 약점

### 문제 1: 모멘텀 급등주를 "비싸다"고 빼버림

대표 사례 — **HMM (2021년)**:
- 해운 운임 폭등 → 실적이 급격히 좋아지는 중
- 회귀 모델: "PBR이 회귀선 위 → 비싸다 → 빼자"
- 실제: HMM은 +29%, +32%씩 계속 상승
- **3번이나** "회귀 때문에 빠진 종목"에 등장

→ 회귀 모델은 **"비싸진 것"**과 **"펀더멘탈이 바뀐 것"**을 구분 못함

### 문제 2: 가치함정(Value Trap)에 빠짐

대표 사례:
- CJ: "싸다"고 편입 → **-22.1%** 하락
- 현대홈쇼핑: "싸다"고 편입 → **-11.6%** 하락
- SBS: "싸다"고 편입 → **-10.1%** 하락

→ **싼 데는 이유가 있는 종목**인데, 회귀 모델은 "왜 싼지"를 모름

### 문제 3: 놓친 기회가 큼

회귀가 손해 본 달에서 종목 수준 분석:
- 회귀로 뺀 종목 중 **상승한 종목: 38개** (놓친 기회)
- 회귀로 뺀 종목 중 하락한 종목: 13개 (잘 뺀 것)

→ 빠진 종목 중 **74%가 상승** — 빼지 말았어야 할 종목을 뺌

## 슬라이드 8: 회귀 약점 종합 — 남대표님 지적과 일치

| 남대표님 지적 | 데이터 확인 결과 |
|-------------|----------------|
| "소외종목이 과집계됨" | ✅ 회귀 매력도 높은 종목 = 시총 작고 관심 없는 종목 |
| "전체 시장을 대표하지 못함" | ✅ 회귀Only 시총이 기존전략 대비 매우 낮음 |

### 회귀 모델의 근본적 한계

```
지금 회귀 모델이 하는 일:
  "이 종목 PBR이 ROE 대비 싸네" → 매력적! → 끝

모르는 것:
  ❌ 왜 싼지 (구조적 문제 vs 일시적 저평가)
  ❌ 언제 오를지 (타이밍)
  ❌ 시장이 지금 어떤 상황인지 (레짐)
```

## 슬라이드 9: 개선 방향 — Trading Agent 제안

### 현재 vs 제안

| | 현재 (정적 회귀) | 제안 (Trading Agent 보완) |
|---|---|---|
| 판단 시점 | 월 1회 스냅샷 | **이벤트 기반** (실적 발표, 뉴스, 수급 변화) |
| 종목 평가 | "싸다/비싸다" 이분법 | **"왜 싼지"** 확인 (공시/뉴스 분석) |
| 시장 상황 | 무시 | **레짐 판단** (상승/하락/횡보) → 회귀 비중 동적 조절 |
| 모멘텀 구분 | 못함 | **"진짜 비싼 것" vs "펀더멘탈 변화"** 구분 |

### 구체적으로 Agent가 하는 일

1. **가치함정 필터** — "이 종목이 싼 이유"를 공시/뉴스에서 확인
   - 구조적 문제 (사업 축소, 경쟁 심화) → 편입 차단
   - 일시적 이벤트 (일회성 비용) → 편입 허용

2. **모멘텀 구분** — 회귀선 위 종목이 "진짜 비싼 건지" 확인
   - 실적이 바뀌고 있으면 → 빼지 않음 (HMM 사례 방지)
   - 실적 변화 없이 주가만 올랐으면 → 빼기

3. **회귀 비중 동적 조절**
   - 하락장에서는 회귀 비중 높임 (회귀가 방어적)
   - 상승장에서는 회귀 비중 낮춤 (모멘텀 놓치지 않도록)

## 슬라이드 10: 지표 설명 (Q&A)

### MDD (Maximum Drawdown)
- 백테스트 기간 중 **고점 대비 가장 많이 빠진 비율**
- 계산: 매 시점에서 역대 최고점 대비 하락률 → 그 중 최대값
- 월별 리밸런싱 시점 기준 (일중 변동은 미반영)

### 평균 회전율
- 매월 리밸런싱 시 **(신규 편입 + 편출) / (2 × 전체 종목 수)**
- 회전율 높으면 거래비용 증가 → 백테스트에서 편도 30bp 반영

### HHI (Herfindahl-Hirschman Index)
- **포트폴리오 집중도** 지표: 각 종목 비중(%)을 제곱하여 합산
- 30종목 균등이면 HHI ≈ 333 / 높을수록 소수 종목에 집중

### 포트폴리오 비중 산정
1. 30개 종목 선정 (팩터 점수 기준)
2. **시가총액 비례**로 초기 비중 배분
3. **10% 캡** 초과분을 나머지에 재배분 (반복)

### 승률 (롤링 24개월)
- 24개월 윈도우를 이동하며 **KOSPI200 대비** 초과수익 양수인 구간의 비율
- 두 전략 간 비교가 아닌, 각 전략 vs 벤치마크

### IS/OOS (In-Sample / Out-of-Sample)
- 모델이 **과적합(찍어맞추기)인지 검증**하는 방법
- IS: 모델이 학습한 기간 → 성과 좋은 게 당연
- OOS: 모델이 못 본 기간 → 여기서도 좋아야 진짜
